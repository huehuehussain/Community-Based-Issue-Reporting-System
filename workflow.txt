Workflow 1: Citizen Issue Reporting & Community Validation

Workflow 2: Authority/Admin Review & Status Management
Actor: Admin / Government Authority

Step-by-Step Flow

Admin logs in through a secure admin panel.

System verifies admin credentials and role.

Admin accesses the admin dashboard.

Dashboard displays all reported issues with:

Category

Location

Priority

Current status

Admin filters issues by:

Area

Category

Priority

Date

Admin selects a specific issue.

System displays:

Issue description

Uploaded images

Upvote count

Reporting timestamp

Admin reviews the issue and takes action.

Admin updates issue status:

Pending → In Progress

In Progress → Resolved / Rejected

Admin adds remarks or resolution notes.

System records:

Status change timestamp

Responsible authority

System automatically notifies:

Original reporter

Users who upvoted the issue

Resolved issues remain visible for transparency.

System Outcomes

Clear accountability

Traceable response history

Citizen trust through updates
